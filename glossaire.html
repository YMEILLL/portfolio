Glossaire & Explications Techniques

Explications simples des termes techniques et logiciels utilisés dans mes missions. Ces termes sont référencés en bleu et soulignés sur les pages de missions.

Concepts Réseau

Routeur

Appareil ou logiciel qui transfère les données entre différents réseaux. Il utilise une table de routage pour décider où envoyer chaque paquet. Un routeur peut être physique (serveur) ou virtuel (logiciel).

Routage Statique

Configuration manuelle des chemins (routes) que les données doivent suivre. L'administrateur réseau décide explicitement où envoyer les paquets. Peu flexible, mais simple et prévisible.

Routage Dynamique

Les routeurs découvrent automatiquement les meilleurs chemins vers chaque réseau grâce à des protocoles (IS-IS, BGP). Si une liaison tombe, le protocole trouve une alternative. Flexible et résilient.

IS-IS (Intermediate System to Intermediate System)

Protocole de routage dynamique qui fonctionne en deux niveaux (Link-State). Chaque routeur connaît la topologie complète du réseau et calcule les meilleurs chemins. Très utilisé dans les infrastructures opérateur (Orange, etc.).

BGP (Border Gateway Protocol)

Protocole de routage utilisé entre domaines réseau différents (AS). C'est le protocole d'Internet. Chaque AS annonce les réseaux qu'il gère, et BGP calcule les meilleurs chemins entre AS.

Link-State (État des Liaisons)

Type de protocole de routage où chaque routeur partage l'état de ses connexions directes. Tous les routeurs construisent une vision complète du réseau. Exemple : IS-IS, OSPF.

Topologie Réseau

La structure d'un réseau : comment les routeurs et liaisons sont connectés entre eux. Exemple : 6 routeurs avec 5 liaisons formant une chaîne ou un anneau.

Interface Réseau

Port de connexion sur un routeur ou serveur. Chaque interface a une adresse IP et peut être connectée à une liaison. Un routeur a plusieurs interfaces pour communiquer avec d'autres appareils.

Outils & Logiciels

EVE-NG (Emulated Virtual Environment - Next Generation)

Simulateur graphique de réseaux. Vous créez des topologies en glissant-déposant des routeurs virtuels et en les reliant. Interface web intuitive. Parfait pour apprendre les protocoles réseau.

Containerlab

Outil d'automatisation pour créer des topologies réseau avec des routeurs Docker. Vous définissez la topologie en fichier YAML, et Containerlab lance automatiquement tous les conteneurs et les relie. Infrastructure as Code pour les réseaux.

Docker

Plateforme de conteneurisation. Empaquete une application et ses dépendances dans un conteneur léger et portable. Permet de lancer plusieurs instances rapidement sans virtualisation lourde.

Conteneur Docker

Instance légère d'une application empaquetée. Contrairement aux machines virtuelles, les conteneurs partagent le noyau du système hôte, d'où leur légèreté. Peut démarrer en secondes.

Image Docker

Modèle ou template pour créer un conteneur. Contient le système d'exploitation, les dépendances et l'application. Une image = plusieurs conteneurs possibles.

YAML (YAML Ain't Markup Language)

Format de fichier lisible par l'humain pour structurer des données. Utilise l'indentation pour créer une hiérarchie. Très utilisé en DevOps pour configurer des services (Containerlab, Docker Compose, Kubernetes).

Bash (Bourne Again Shell)

Langage de script pour Linux/Unix. Permet d'automatiser des tâches en écrivant des commandes. Essentiellement une succession de commandes terminal exécutées automatiquement.

Wireshark

Analyseur de trafic réseau. Capture et affiche tous les paquets qui passent sur une liaison. Permet d'inspecter le contenu et le timing des échanges pour déboguer des problèmes réseau.

SSH (Secure Shell)

Protocole sécurisé pour accéder à distance à un serveur. Chiffre la connexion et utilise des clés pour s'authentifier. Remplace le Telnet non sécurisé. Standard de l'administration Linux.

Bastion SSH

Serveur intermédiaire sécurisé pour accéder à des serveurs sensibles. Vous vous connectez d'abord au Bastion, puis de là au serveur cible. Réduit les risques de sécurité en contrôlant les accès.

Infrastructure & Systèmes

Dell PowerEdge R730

Serveur physique professionnel. Dans mon cas : 56 Go RAM, 2 processeurs Xeon E5. Capable de faire tourner 10-15 routeurs Docker simultanément. Installé en baie dans un datacenter.

Ubuntu Server

Système d'exploitation Linux basé sur Debian. Version serveur sans interface graphique, axée sur la ligne de commande. Léger, stable et très utilisé en production.

Linux

Noyau open-source (cœur du système d'exploitation). Utilisé dans la plupart des serveurs. Permet un contrôle bas niveau du matériel et une automatisation complète.

Datacenter

Centre de données. Bâtiment sécurisé avec serveurs physiques en baies. Connexions Internet haut débit, alimentation électrique redondée, climatisation. Lieu où tournent les services en production.

Bare Metal

Serveur sans virtualisation : le système d'exploitation tourne directement sur le matériel. Pas de couche hyperviseur. Plus de performance, mais moins de flexibilité qu'une VM.

RAID (Redundant Array of Independent Disks)

Technique pour stocker les données sur plusieurs disques simultanément. Offre de la redondance : si un disque tombe, les données sont toujours accessibles. Sécurité de stockage.

iDRAC (Integrated Dell Remote Access Controller)

Système de gestion à distance sur les serveurs Dell. Permet d'accéder au serveur hors bande (indépendamment de l'OS), de le redémarrer ou de surveiller sa santé.

Concepts DevOps & Automatisation

Infrastructure as Code (IaC)

Traiter l'infrastructure (serveurs, réseaux, configurations) comme du code. Au lieu de configurer manuellement, on décrit tout dans des fichiers (YAML, scripts) qui sont versionnés et reproductibles.

Automatisation

Exécuter automatiquement une série de tâches sans intervention humaine. Les scripts Bash lancent des commandes répétitives. Gagne du temps, réduit les erreurs.

Git

Système de versioning. Permet de tracker les modifications de fichiers, de collaborer en équipe et de revenir à une version antérieure si nécessaire. Essentiel pour IaC.

CI/CD (Continuous Integration / Continuous Deployment)

Pipeline automatisé. À chaque push de code, les tests s'exécutent automatiquement et le déploiement se fait sans intervention manuelle. Gagne du temps et améliore la qualité.

DevOps

Culture et ensemble de pratiques qui unissent développeurs et opérateurs. Utilise l'automatisation, l'IaC et les pipelines CI/CD pour déployer rapidement et fiablement.

iPerf (iPerf3)

Outil pour tester les performances d'un réseau. Génère du trafic entre deux points et mesure la bande passante, la latence et les pertes. Utilisé pour valider les topologies.

Sécurité

Chiffrement

Technique pour rendre les données illisibles à quiconque ne détient pas la clé. SSH chiffre les commandes qu'on envoie au serveur. Essentiel pour protéger les données sensibles.

Clé Publique / Privée

Paire de clés utilisée en cryptographie asymétrique. La clé publique peut être partagée, la clé privée reste secrète. Permet d'authentifier sans envoyer de mot de passe (SSH).

Authentification

Vérifier l'identité d'une personne ou d'un service. Par exemple, SSH authentifie via clés ou mot de passe. Bastion contrôle qui peut accéder aux serveurs sensibles.

Accès à Distance Sécurisé

Pouvoir administrer un serveur depuis un autre endroit, de manière chiffre et authentifiée. SSH et Bastion permettent ça en production tout en minimisant les risques.
