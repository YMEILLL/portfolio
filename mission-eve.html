Mission 1 : Apprentissage du Routage sur EVE-NG

Contexte de départ

Pour appréhender les architectures réseaux d'Orange, j'ai commencé par réaliser des maquettes sur EVE-NG, hébergé sur le serveur d'un collègue. Cet environnement partagé m'a permis de manipuler des équipements virtuels (Cisco, Juniper) sans contrainte matérielle immédiate.

L'objectif était purement pédagogique : comprendre le fonctionnement des protocoles de routage (IS-IS) avant d'envisager leur automatisation.

Environnement Technique

  • Infrastructure : Serveur EVE-NG partagé
  • Protocole : IS-IS (Link-State)
  • Outils : Wireshark, CLI Cisco/Juniper
  • Limite : Configuration 100% manuelle.

Les Limites (Transition vers la suite)

Bien que cet environnement soit fonctionnel, j'ai identifié deux freins majeurs à l'industrialisation de mes tests :

  1. Manque d'autonomie : Travailler sur le serveur d'un collègue créait une dépendance et ne me permettait pas d'avoir la main sur l'administration système (OS, Docker).
  2. Absence d'automatisation : EVE-NG est un outil "clic-bouton". Il est très difficile d'automatiser le déploiement et les tests via des scripts (CI/CD), ce qui était l'objectif fixé par mon tuteur.

➔ Décision : Migrer vers une solution "Infrastructure as Code" (Containerlab) sur une machine dont j'aurais la pleine maîtrise.
